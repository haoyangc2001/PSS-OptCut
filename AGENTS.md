# Repository Guidelines

## 项目结构与模块组织
- `src/`: 核心 Python 代码。
  - `src/main.py`: 入口脚本，循环生成随机实例、调用求解并汇总结果。
  - `src/Pamas_generator.py`: 参数/实例生成器（客户、服务等级、时间窗、成本等）。
  - `src/Solver_builder.py`: Gurobi 模型构建与求解逻辑。
  - `src/sample_generator.py`: 训练样本打包与统计汇总。
- `data/`: 数据文件与结果产物（CSV/XLSX）。
- `notebooks/`: 分析与实验用 Notebook。
- `document/`: 业务需求与说明文档。

## 构建、运行与开发命令
- `python src/main.py`: 运行完整实例生成 + 优化求解流程。
- `python -m pip install gurobipy numpy pandas openpyxl`: 安装核心依赖（Gurobi 需要有效许可证）。
- `jupyter notebook`: 打开 `notebooks/` 目录下的分析笔记。

## 编码风格与命名规范
- 缩进：4 空格，避免使用 Tab。
- 命名：类使用 `CamelCase`，函数/变量使用 `snake_case`，常量用 `UPPER_CASE`。
- 参数显式传递为主，优先保证可读性。

## 测试指南
- 目前无自动化测试。若补充测试，请新增 `tests/` 并在文档中说明运行方式（如 `pytest`）。
- 结果复现建议记录 `random_seed` 与关键参数区间。

## 依赖与配置提示
- 依赖：NumPy、Pandas、OpenPyXL、Gurobi（`gurobipy`）。
- `src/main.py` 默认将结果追加写入 `D:/桌面/result.csv`（Windows 路径），在非 Windows 环境需调整为本机有效路径。

## 创新点
- 有效不等式（valid cut）定制与分阶段添加，用于提升求解可行性与效率，并对不同切割组合进行系统评估。
- 数据驱动策略：记录“实例特征 + 求解表现”形成数据集，用于预测求解时间/策略选择，从而满足业务时效性需求。

## 提交与合并请求
- 当前目录不是 Git 仓库，无法读取历史提交规范。
- 如需提交，建议使用简洁的祈使句（如“优化求解回调逻辑”），PR 需说明复现命令与数据变更。

## 项目摘要（用于汇报/论文）
本项目聚焦于产品-服务系统（PSS）配置与交付的联合优化问题，将客户服务选择、工程团队路线与时间窗约束统一建模为混合整数规划（MILP）。在满足库存、工时、路线连续性等业务硬约束的前提下，通过可配置的多目标权重平衡客户效用、服务及时性与企业利润。为提高求解效率与可行性，项目定制有效不等式并采用分阶段添加策略，系统评估不同切割组合的性能影响。同时，项目记录大量实例特征与求解表现，构建数据集以支持机器学习预测求解时间与策略选择，满足业务对时效性的要求并提升决策效率。
